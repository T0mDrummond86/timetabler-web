# Deploy on Render (Blueprint)

One-click deploy using [Render Blueprints](https://render.com/docs/blueprint-spec). Creates a free Postgres database, a Docker API service, and a static frontend with SPA routing.

## Prerequisites

- GitHub repo with **timetabler-web** pushed to `main`
- [Render](https://render.com) account (free tier is enough to try)

## Steps

1. In Render: **New → Blueprint**.
2. Connect the `timetabler-web` repository.
3. Render reads [`render.yaml`](../render.yaml) and shows three resources:
   - `timetabler-db` — Postgres
   - `timetabler-api` — FastAPI (Docker, runs Alembic on start)
   - `timetabler-web` — Vite static site
4. Click **Apply**. First deploy takes several minutes (npm build + Docker image).
5. When green, open the **timetabler-web** URL (e.g. `https://timetabler-web.onrender.com`).
6. **Register** the first admin user while `ALLOW_REGISTRATION=true` (default in blueprint).
7. In the **timetabler-api** service → **Environment**, set `ALLOW_REGISTRATION=false` and redeploy.

## What the blueprint sets

| Variable | Service | Source |
|----------|---------|--------|
| `DATABASE_URL` | API | Render Postgres `connectionString` (auto-normalized to `postgresql+psycopg://`) |
| `JWT_SECRET` | API | Generated by Render |
| `CORS_ORIGINS` | API | Frontend `RENDER_EXTERNAL_URL` |
| `VITE_API_URL` | Frontend (build) | API `RENDER_EXTERNAL_URL` |
| `ENVIRONMENT` | API | `production` |
| `AUTO_CREATE_TABLES` | API | `false` |

## Smoke test

```bash
curl -s "https://YOUR-API.onrender.com/health"
# {"status":"ok",...}

curl -s -o /dev/null -w "%{http_code}" "https://YOUR-API.onrender.com/docs"
# 404 in production
```

Open the frontend, log in, create a session, import a small file.

## Using Neon instead of Render Postgres

1. Create a project on [Neon](https://neon.tech) and copy the connection string.
2. Remove the `databases:` block from `render.yaml` (or do not apply it).
3. On **timetabler-api**, set `DATABASE_URL` manually to your Neon URL (`postgresql://…` is fine).
4. Redeploy the API.

## Local production image test

From repo root:

```bash
docker build -f backend/Dockerfile.prod -t timetabler-api:prod .
docker run --rm -p 8000:8000 \
  -e DATABASE_URL=postgresql+psycopg://timetabler:timetabler@host.docker.internal:5432/timetabler \
  -e JWT_SECRET=local-prod-test-secret-not-a-placeholder-min-32-chars \
  -e ENVIRONMENT=production \
  -e AUTO_CREATE_TABLES=false \
  -e CORS_ORIGINS=http://localhost:5173 \
  timetabler-api:prod
```

Requires Postgres reachable at the URL (e.g. `docker compose up postgres -d`).

## Notes

- Free web services **spin down** after inactivity; first request may take ~30s.
- Free Postgres expires after 90 days unless upgraded; export backups before then.
- SPA deep links work via `routes` in `render.yaml` and `frontend/public/_redirects`.

See also [`PRODUCTION_CHECKLIST.md`](PRODUCTION_CHECKLIST.md).
